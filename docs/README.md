# Техническая документация LexGuard

Добро пожаловать в раздел документации проекта **LexGuard** — интеллектуальной системы автоматизированного юридического анализа контрактов.

Этот раздел содержит актуальную информацию о внутренней архитектуре проекта, процессах машинного обучения, потоках данных и выявленных рисках/краевых случаях. Документация предназначена для разработчиков, желающих восстановить контекст или продолжить развитие системы.

## Навигация по документации

Ниже представлен перечень основных документов проекта:

- 🏗️ **[Архитектура системы (ARCHITECTURE.md)](ARCHITECTURE.md)**
  Описывает общую структуру проекта, взаимодействие компонентов (FastAPI, Nginx, PostgreSQL, Celery, Redis, Qdrant, Ollama) и схему базы данных.

- 🧠 **[RAG и бизнес-логика (RAG_AND_LOGIC.md)](RAG_AND_LOGIC.md)**
  Подробное описание работы с векторной базой `Qdrant`, логики поиска эталонных норм (с учетом типа договора и пороговых значений), процессов LLM (от определения типа документа до парсинга JSON) и группировки выявленных рисков.

- 🔄 **[API и потоки данных (API_AND_DATA_FLOW.md)](API_AND_DATA_FLOW.md)**
  Документирует основные эндпоинты (включая загрузку и `/analyze/{id}/grouped`) и схему обмена данными между клиентской частью (Vue/Vanilla JS) и бэкендом (Celery + Redis polling).

- ⚠️ **[Краевые случаи и риски (EDGE_CASES_AND_RISKS.md)](EDGE_CASES_AND_RISKS.md)**
  Анализ текущих уязвимостей в логике, хардкодных лимитов и фоллбеков. Описаны места, где система может вести себя непредсказуемо, и предложены пути их устранения.

- 📖 **[Схема правовых норм (LEGAL_NORMS_SCHEMA.md)](LEGAL_NORMS_SCHEMA.md)**
  Описание канонической структуры JSON-объекта для эталонной правовой нормы, используемой в RAG.

## Результаты последнего аудита (Апрель 2026)
В ходе ревизии документации текущее состояние кода было сверено с документами. Основные изменения:
- Добавлено детальное описание логики RAG и API.
- Удалены неактуальные документы (`ROADMAP_PRIORITIES.md`, `DEFENSE_DEMO_SCRIPT.md`), так как их задачи уже выполнены или они не несут технической ценности.
- Актуализировано описание моделей, эндпоинтов и процесса классификации типов контрактов в соответствии с фактическим кодом (`analyzer.py`, `rag.py`).
